# Quick Fix Command

For simple, obvious fixes that don't need research or planning.

## When to Use

✅ Fixing typos
✅ Updating dependencies
✅ Small bug fixes with known solutions
✅ Formatting/linting fixes
✅ Simple documentation updates

## When NOT to Use

❌ Complex features
❌ Architectural changes
❌ Anything affecting multiple files significantly
❌ Bugs with unknown causes
❌ Changes that need testing strategy

## Instructions

1. Ask: "What needs to be fixed?"
2. Understand the fix needed
3. Make the change
4. Verify it works (run tests if applicable)
5. Commit with descriptive message
